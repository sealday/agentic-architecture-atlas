# Sidebar Navigation Refresh Design

## Goal

Make the documentation sidebar communicate the site hierarchy clearly on narrow
screens without replacing Docusaurus navigation behavior.

## Problem

The current sidebar wraps an autogenerated directory inside a manually declared
category. Docusaurus therefore renders each directory index as a normal child:
`案例库` appears among case articles, while single-page sections expand to a
child with the same label. Long case titles also combine the subject and
explanatory subtitle, making the navigation dense and difficult to scan.

## Considered approaches

1. **Restore one root autogenerated sidebar — selected.** Docusaurus recognizes
   each directory `index.mdx` as its category link, removes that document from
   the children, and flattens categories that contain no other pages. This keeps
   future content automatic and uses framework-supported accessibility.
2. **Maintain every sidebar item manually.** This gives exact ordering and
   labels but requires editing configuration whenever content changes.
3. **Build a custom sidebar component.** This permits richer search and custom
   metadata but introduces unnecessary component, state, and upgrade surface
   for the current problem.

## Design

- Replace the manually nested category configuration with one root
  `autogenerated` sidebar.
- Use each section index as its category link. Sections with only an index page
  become direct links rather than expandable duplicate rows.
- Give section indexes explicit `sidebar_position` values so the order remains
  首页、案例库、架构模式、设计题、学习路径、资料库.
- Give case documents concise `sidebar_label` values containing the case subject
  only. Full descriptive titles remain unchanged on pages and in the catalog.
- Use CSS to reinforce hierarchy: distinct category weight, consistent
  indentation, larger row hit areas, a stable active marker, readable contrast,
  and single-line ellipsis with native title/tooltips left to Docusaurus.
- On narrow screens, keep the site title and menu control visible while hiding
  duplicated top-level navbar links from the mobile drawer header area.

## Behavior and accessibility

- Category rows remain native Docusaurus links/buttons and retain keyboard,
  focus, expanded-state, and screen-reader behavior.
- Active navigation uses more than color: background, weight, and a left marker.
- Focus styling remains visible.
- Motion follows the existing reduced-motion rule.
- No content URLs or page titles change.

## Verification

- Add a regression test that asserts the root autogenerated configuration,
  explicit section ordering, concise case labels, and removal of manual duplicate
  category wrappers.
- Run the focused regression test through a red-green cycle.
- Run the full test suite, content validation, catalog check, typecheck, and
  production build.
- Inspect desktop and narrow-screen screenshots from the production build.
- Push the verified commit to `main` and confirm the GitHub Pages workflow and
  deployed page.

## Scope

This change does not add sidebar search, alter page content, or introduce a
custom navigation component. The untracked in-progress Kong case is not modified
or included in the commit.
