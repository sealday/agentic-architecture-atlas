# Sidebar Navigation Refresh Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove duplicate sidebar nodes and make the documentation hierarchy readable and usable on narrow screens.

**Architecture:** Let Docusaurus generate the complete sidebar from the content root so directory index documents become category links instead of duplicated children. Keep full document titles unchanged, add concise navigation-only labels through front matter, and reinforce the native menu hierarchy with scoped CSS.

**Tech Stack:** Docusaurus 3.10.2, TypeScript, MDX front matter, CSS, Node.js test runner, GitHub Pages.

## Global Constraints

- Keep Docusaurus native sidebar links, expansion state, keyboard behavior, and screen-reader semantics.
- Do not change content URLs, document titles, or catalog titles.
- Do not add dependencies or a custom sidebar component.
- Preserve the user-authored `content/cases/kong-ai-gateway-routing-resilience.mdx`;
  do not publish it while untracked, and integrate it if the user commits it.
- Publish only after the full `npm run verify` command succeeds.

Execution note: after this plan was committed, `5dac465` added the Kong case to
`main`. The implementation therefore preserves that commit and integrates the
case into the canonical manifest, generated catalog, counts, and concise sidebar
labels instead of treating it as untracked work.

- [ ] **Step 1: Write the failing structural regression test**

Create `tests/sidebar-navigation.test.mjs` with tests that:

```js
import assert from 'node:assert/strict';
import {readFile} from 'node:fs/promises';
import test from 'node:test';

import {requiredCaseSlugs} from '../scripts/content-schema.mjs';

const root = new URL('../', import.meta.url);

async function source(path) {
  return readFile(new URL(path, root), 'utf8');
}

test('uses one root autogenerated sidebar instead of duplicate manual categories', async () => {
  const sidebar = await source('sidebars.ts');

  assert.match(sidebar, /type:\s*'autogenerated'/);
  assert.match(sidebar, /dirName:\s*'\.'/);
  assert.doesNotMatch(sidebar, /label:\s*'案例库'/);
  assert.doesNotMatch(sidebar, /dirName:\s*'cases'/);
});

test('orders section index links without rendering duplicate children', async () => {
  const expected = new Map([
    ['content/intro.mdx', 1],
    ['content/cases/index.mdx', 2],
    ['content/patterns/index.mdx', 3],
    ['content/questions/index.mdx', 4],
    ['content/paths/index.mdx', 5],
    ['content/references/index.mdx', 6],
  ]);

  for (const [path, position] of expected) {
    assert.match(await source(path), new RegExp(`sidebar_position: ${position}`));
  }
});

test('gives every canonical case a concise navigation-only label', async () => {
  for (const slug of requiredCaseSlugs) {
    const filename = slug.split('/').at(-1);
    const content = await source(`content/cases/${filename}.mdx`);
    const label = content.match(/^sidebar_label: (.+)$/m)?.[1];

    assert.ok(label, `Missing sidebar_label for ${slug}`);
    assert.ok(label.length <= 42, `Sidebar label is too long for ${slug}`);
    assert.doesNotMatch(label, /[：:]/, `Sidebar label contains a subtitle for ${slug}`);
  }
});
```
