# Tego Arch Hand-Drawn Visual Language

Use this reference to create a consistent family of original architecture illustrations. It distills general visual traits from the supplied examples; it does not authorize copying their character, signature, watermark, wording, or composition.

## Visual identity

| Dimension | Contract |
| --- | --- |
| Canvas | Warm white or very light cream paper, 16:9 landscape, generous outer margins. |
| Line | Slightly imperfect marker or felt-tip line; crisp enough for documentation, never sketchy to the point of ambiguity. |
| Palette | Deep blue for the main path, orange for decisions/emphasis, green for success, red only for failure, charcoal for body text. |
| Typography | Large handwritten-style title; compact Chinese labels; English technical terms only when they are canonical identifiers. |
| Shapes | Rounded rectangles, simple cards, hand-drawn arrows, restrained dashed boundaries, familiar technical icons. |
| Density | One dominant reading path, 4–7 major groups, 6–14 generated labels, ample whitespace. |
| Finish | Educational whiteboard infographic with editorial polish; flat 2D, no photorealism or glossy 3D. |

## Composition grammar

Choose the smallest grammar that proves the article's point:

- **Left-to-right flow:** request, handoff, processing, result, failure return.
- **Circular loop:** observe, decide, act, receive result, decide again.
- **Swimlane:** distinct control owners or parallel workers.
- **State map:** normal path in blue/green, failure and retry paths in red/orange.
- **Layer stack:** memory, control, storage, or responsibility layers.
- **Comparison board:** two or more aligned columns with the same row questions.

Place the title at the top. Put the main diagram in the center. Reserve the lower band for one concise conclusion. A small generic guide figure may appear only when it adds orientation; create a new project-neutral figure and never reproduce a reference person's likeness, clothing formula, pose, name, or signature.

## Semantic color rules

- Blue: primary architecture, normal control flow, neutral structure.
- Orange: decision, emphasis, cost, optional or conditional path.
- Green: completed, accepted, healthy, safe outcome.
- Red: failed, rejected, retry exhausted, hazardous path.
- Charcoal: explanation that carries no state semantics.

Never rely on color alone. Pair state color with a label, icon, line style, or shape.

## Text rules

Use exact simplified Chinese. Keep labels to 2–8 Chinese characters when possible. Preserve code identifiers verbatim. Do not invent a handwritten substitute for an uncertain term.

Prefer:

- title: one line;
- subtitle: optional one-line equation or scope;
- node labels: nouns or verb-object phrases;
- conclusion: one sentence under 24 Chinese characters.

Avoid paragraphs, footnotes, URLs, version inventories, code listings, small tables, decorative English, and any watermark or signature.

## Originality boundary

Reference images may inform medium, palette, density, and educational tone. They must not supply:

- an identifiable person or mascot;
- a signature, copyright line, watermark, or social handle;
- exact title wording or sentence rhythm;
- traced icons, spatial arrangement, or a distinctive sequence of panels;
- factual content not supported by the article.

Describe the style through generic attributes. Do not name or emulate a living artist.
