# 案例库

案例库用统一模板拆解真实项目，避免把框架术语、公开事实与作者判断混在一起。每篇案例依次说明：问题与约束、角色与职责、控制流与数据流、通信协议、状态和故障恢复、质量属性、代价、未确认事项，以及可回查的一手来源。

## 证据标签

- **已证实事实**：官方文档、官方仓库或可复查实现直接支持的陈述。
- **基于证据的推断**：多个事实共同指向、但来源没有直接表述的架构结论；必须写明推断链和不确定性。
- **个人分析**：用于比较、迁移或提出设计建议的作者判断；不能冒充项目原意。

## 筛选维度

每篇案例都用 `domains`、`agent_patterns`、`protocols`、`quality_attributes`、`difficulty`、`status`、`confidence`、`analyzed_at` 与 `source_cutoff` 描述可筛选元数据。读者可以据此按领域、控制模式、协议、质量目标、阅读难度和证据新鲜度横向比较，而不必只依赖产品名称。
