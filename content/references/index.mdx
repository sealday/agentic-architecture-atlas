# 资料库

资料库保存案例判断所依赖的证据入口，而不是堆积未经分级的链接。每条来源同时公开它能够支持的证据角色、不能单独证明的内容、许可证与版权处理方式，以及站内实际使用位置。

## 如何使用资料库

来源按证据强度分为四层：

1. **一手来源**：标准、官方文档、官方仓库、论文与可检查源码，用于确认协议语义、公开 API、实现结构和明确陈述。
2. **第一方工程资料**：维护团队或厂商发布的工程博客、事故报告与参考架构；可支持其自身系统和实践的公开事实，但厂商主张仍需与本站判断分开。
3. **可信二手来源**：教材和独立分析，用于比较、历史背景与学习，不作为高风险运行事实的唯一依据。
4. **发现与导航**：社区索引和路线图只帮助选题与定位原始资料，不是事实证据。

引用来源并不自动证明文章中的全部结论。写作与评审仍须区分来源直接支持的事实、跨来源推断和个人分析。

## 索引不等于证据

社区索引只用于选题和学习导航。索引收录某个项目或观点，不代表项目已通过安全、质量或生产验证；事实结论必须回到原作者、官方文档、标准、论文或可检查实现。

## 版权处理

| 页面标记 | 使用方式 |
| --- | --- |
| 仅提炼事实与短引文 | 只总结可验证事实；短引文保持必要长度并附署名。 |
| 允许改编，必须署名 | 按许可证要求标明作者、来源和修改情况。 |
| 改编须署名、相同方式共享并复核 | 除署名外，复核衍生作品的相同方式共享义务。 |
| 厂商主张与本站判断分开 | 明确哪些内容是厂商公开陈述，哪些是本站分析。 |
| 本站原创 | 仅用于本站原创结构、文字或插图，不扩张到第三方材料。 |

## 链接策略

`stable` 表示预期长期可访问的固定入口；`floating` 表示会随最新版本移动的入口；`auth-required` 表示访问需要身份；`retired` 表示来源已退役并须保留替代关系。链接状态将在 Task 6 的可重复检查接入后显示；当前占位不代表已完成在线健康检查。

## 如何登记来源

新增或修改来源时，只编辑规范化来源账本并运行内容生成与治理校验。登记项必须包含作者或机构、来源类型与层级、版本、核查日期、许可证范围和证据、版权处理、允许的证据角色、使用边界，以及每个使用文档的署名说明。文档路由来自已验证 metadata，不从文件名猜测。
